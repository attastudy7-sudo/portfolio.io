# Hero Section Animation Fix Plan

## Problem Analysis
The hero section animation is not working because:
1. The stylesheet was accidentally overwritten with only hero section content
2. The full original stylesheet needs to be restored from git history
3. Then the specific hero animation fix needs to be applied correctly

## Root Cause
In a previous edit, I used the write tool incorrectly, replacing the entire `style.css` file with only the hero section styles instead of making a targeted edit to fix the animation issue.

## Steps to Fix
1. Restore the full original stylesheet from git history (HEAD~1)
2. Apply the correct fix for the hero animation will-change property override issue
3. Verify the fix works properly

## Detailed Fix Required
The actual issue needing to be fixed is in the CSS where a shared rule was overriding the `will-change` property:

Likely problematic code in original stylesheet:
```css
body.hero-ready .hero-animate,
body.js-scroll-reveal .is-visible {
    will-change: auto; /* This overrides the hero-animate will-change */
}
```

Should be changed to:
```css
body.js-scroll-reveal .is-visible {
    will-change: auto;
}
/* body.hero-ready .hero-animate should NOT override will-change */
```

Or alternatively, explicitly maintain the correct will-change for hero-animate:
```css
body.hero-ready .hero-animate {
    will-change: transform, opacity !important;
}
```

## Verification
After fix, hero elements should:
1. Start with opacity: 0 and transform: translateY(26px) (plus element-specific offsets)
2. Transition to opacity: 1 and transform: none when hero-ready class is added
3. Will-change property properly set for smooth animation performance
